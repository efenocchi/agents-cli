Notes for picking a direction

If creation date is the only signal we trust: ship Variant A (tiered text) + B (ribbon). Zero new data, biggest visual lift. Two lines of conditional CSS in TaskCard.tsx.
If we pull dueDate through to the webview: Variant C is the right answer. The data already lives in extension/src/core/tasks.ts line 26 (dueDate) and extension/src/vscode/linear.vscode.ts line 85 — it's just not on the UnifiedTask.metadata shape in extension/ui/settings/types/index.ts yet. Adding it is ~5 lines.
For "carried over N cycles": requires fetching cycle on each issue from Linear (not just the active cycle). Roughly: extend the GraphQL query in linear.vscode.ts, store the cycle number on metadata, compare against active cycle to compute carryover count.
Recommended ship sequence: A → B → C → D. Each step is independent and improves the signal without rewriting the card.
